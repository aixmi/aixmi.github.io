## 说明

本规范的目的是为了统一组内或者公司内部的编码风格。这样做有几个好处：

- 有利于相互之间check代码，减少由于不同编码风格带来的沟通成本
- 规范往往是从实际的业务中总结出来的，遵守规范可以让我们在业务落地的时候少走技术的弯路
- 缩小不同水平的技术人员之间的代码质量差距
- 学到一些最佳实践

本规范并不能满足所有业务或者公司的需求，建议参考此规范形成符合自身需求的文档。
